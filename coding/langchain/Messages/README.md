# Messages Chapter Code Examples

This directory contains Python code examples demonstrating LangChain's message system, which is the fundamental unit of context for models.

## Files

| File | Description |
|------|-------------|
| `llm_config.py` | LLM configuration for all examples |
| `01_basic_usage.py` | Basic usage of LangChain messages with SystemMessage and HumanMessage |
| `02_text_prompts.py` | Using text prompts for simple generation tasks |
| `03_message_prompts.py` | Using message prompts for multi-turn conversations |
| `04_dictionary_format.py` | Using dictionary format for messages in OpenAI chat completions format |
| `05_system_message.py` | Using SystemMessage to set tone and define model's role |
| `06_human_message.py` | Using HumanMessage with different content types and metadata |
| `07_ai_message.py` | Using AIMessage to represent model responses and create conversation history |
| `08_tool_calls.py` | Using AIMessage with tool calls |
| `09_token_usage.py` | Using AIMessage to access token usage metadata |
| `10_streaming.py` | Streaming responses using AIMessageChunk |
| `11_tool_message.py` | Using ToolMessage to pass tool execution results back to the model |
| `12_tool_message_artifact.py` | Using ToolMessage with artifact field for storing supplementary data |

## Key Concepts

### Message Types
- **SystemMessage**: Instructions that prime the model's behavior
- **HumanMessage**: Represents user input and interactions
- **AIMessage**: Responses generated by the model, including tool calls and metadata
- **ToolMessage**: Represents the outputs of tool calls

### Message Content
Messages support:
- Text content
- Multimodal content (images, audio, video, files)
- Reasoning blocks
- Tool calls
- Metadata (IDs, names, usage info)

### Usage Patterns
- **Text prompts**: Single strings for simple requests
- **Message prompts**: Lists of message objects for conversations
- **Dictionary format**: OpenAI chat completions format
- **Streaming**: Using `stream()` for real-time responses
- **Tool calling**: Binding tools to models for function execution
