# 什么是提示词链（Prompt Chaining）

根据维基内容，**提示词链（Prompt Chaining）**（也称为**管道模式**）是构建 AI 智能体系统最基础的设计模式，它采用分而治之的策略来处理复杂任务。

## 核心思想

提示词链的核心思想很简单：**不再要求 LLM 在单一的整体化步骤中解决复杂问题，而是将问题分解为一系列更小、更易管理的子问题，每个子问题通过专门设计的提示词单独处理，一个提示词的输出会作为输入传递给链中的下一个提示词**。

这种顺序处理建立了**依赖链**——前一步的输出为后一步提供上下文，使 LLM 能够在先前工作的基础上逐步构建最终解决方案。

## 它解决了什么问题

使用单一复杂提示词处理复杂任务时，往往会出现这些问题：

| 问题 | 说明 |
|------|------|
| **指令忽略** | 部分提示内容被模型忽视 |
| **上下文偏离** | 模型失去对初始上下文的追踪 |
| **错误传播** | 早期错误被后续步骤放大 |
| **上下文窗口不足** | 单一提示词需要容纳所有信息，容易超出限制 |
| **幻觉增加** | 认知负荷增加导致生成更多错误 |

提示词链通过分解问题，每个步骤只专注于一件事，大大降低了 LLM 的认知负荷，从而提高了输出准确性和可靠性。

## 关键原则

1. **分而治之**：将复杂任务分解为多个聚焦的子步骤
2. **顺序依赖**：前一步输出作为后一步输入，建立清晰依赖
3. **模块化**：每个步骤可以独立设计、优化和调试
4. **结构化输出**：使用 JSON/XML 确保步骤间数据传递准确
5. **可扩展性**：支持在步骤之间插入外部工具调用

## 典型应用场景

| 场景 | 工作流 |
|------|--------|
| **信息处理** | 提取文本 → 总结 → 提取实体 → 搜索知识库 → 生成报告 |
| **复杂问答** | 分解子问题 → 分别研究 → 综合信息 → 给出答案 |
| **数据提取** | 尝试提取 → 验证结果 → （如果有错）修正 → 重复直至正确 |
| **内容生成** | 构思主题 → 生成大纲 → 分段起草 → 审查完善 |
| **对话智能体** | 识别意图 → 更新对话状态 → 基于状态生成响应 → 重复 |
| **代码生成** | 理解需求 → 生成大纲 → 编写代码 → 审查改进 → 添加文档 |

## 别名

提示词链也叫**管道模式（Pipeline Pattern）**：
- "提示词链"强调 LLM 提示词的顺序执行
- "管道模式"强调数据流动的结构性视图，数据就像在管道中一步步被处理

## 与其他模式的关系

- 提示词链是**最基础**的智能体设计模式
- **路由**模式在提示词链的基础上添加了**条件动态决策**，使智能体能够根据输入选择不同路径
- 提示词链是构建能够进行多步推理的完整 **AI 智能体** 系统的基础
