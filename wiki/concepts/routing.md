# Routing（路由）

**路由**是一种智能体设计模式，使智能体能够根据条件动态地决定执行路径，将控制流定向到不同的专门处理程序、工具或子智能体。

## 定义

路由模式将条件逻辑引入智能体系统，使其从固定的线性执行路径转变为自适应模型：智能体动态评估特定标准，从一组可能的后续操作中进行选择。这使得系统行为更加灵活且具有上下文感知能力。

## 问题背景

[[concepts/prompt-chaining|提示词链]] 擅长处理确定性、线性工作流，但无法应对需要动态决策的场景。现实世界中的智能体必须根据：
- 用户输入的不同意图
- 当前环境状态
- 前一步操作的结果
- ... 在多个潜在行动之间做出选择

路由提供了这种动态决策能力。

## 四种实现方式

### 1. 基于 LLM 的路由

提示语言模型本身分析输入并输出指示下一步的特定标识符，智能体系统根据输出决定路由。

**示例提示词**：
```
分析以下用户查询并仅输出类别：
"订单状态"、"产品信息"、"技术支持"或"其他"。
```

**优点**：灵活，能够处理语义模糊和新颖输入
**缺点**：增加一次 LLM 调用开销

### 2. 基于嵌入的路由

将输入查询转换为向量嵌入，然后将此嵌入与代表不同路由的嵌入进行比较，查询被路由到嵌入最相似的路由。

**适用场景**：语义路由，决策基于输入的含义而不仅仅是关键词

### 3. 基于规则的路由

使用预定义规则或逻辑（关键词匹配、模式匹配、if-else 语句、switch case）进行决策。

**优点**：快速、确定性、可预测
**缺点**：处理细微或新颖输入的灵活性较差

### 4. 基于机器学习模型的路由

使用在标记数据上训练的判别分类器进行路由决策。路由逻辑被编码在微调模型的权重中，推理时不需要 LLM 生成。

**优点**：推理速度快，适合高吞吐量场景
**缺点**：需要训练数据，不灵活

## 典型应用场景

| 应用领域 | 用例 |
|---------|------|
| **人机交互** | 虚拟助手/AI 导师解释用户意图，选择最合适后续操作 |
| **自动化数据处理** | 分类电子邮件、支持工单，定向到相应工作流 |
| **多智能体系统** | 中央协调器将任务分配给最合适的专门智能体 |
| **工具选择** | 根据用户查询选择调用哪个工具 |

## 框架支持

- **[[concepts/langgraph|LangGraph]]**：基于状态的图架构特别适合复杂路由场景，决策可以依赖整个系统的累积状态
- **Google ADK**：提供自动路由机制，框架将用户意图匹配到正确的工具/子智能体
- **[[concepts/langchain|LangChain]]**：提供 `RunnableBranch` 组件支持条件路由

## 何时使用

当满足以下条件时使用路由模式：
- ✅ 智能体必须根据用户输入或当前状态在多个不同路径间决策
- ✅ 需要处理各种不同类型的请求
- ✅ 需要超越固定顺序的线性执行
- ✅ 系统需要自适应响应不同输入
