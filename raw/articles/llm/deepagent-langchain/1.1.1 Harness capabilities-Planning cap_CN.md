Agent harness 是多种不同功能的组合，使得构建长时间运行的 Agent 变得更加容易：

除了这些功能外，Deep Agents 还使用 [Skills](#skills) 和 [Memory](#memory) 来提供额外的上下文和指令。

## 规划能力

harness 提供了一个 `write_todos` 工具，Agent 可以使用它来维护结构化的任务列表。

**功能：**

- 跟踪多个任务及其状态（`'pending'`、`'in_progress'`、`'completed'`）
- 持久化存储在 Agent 状态中
- 帮助 Agent 组织复杂的多步骤工作
- 适用于长时间运行的任务和规划

## 虚拟文件系统访问

harness 提供了一个可配置的虚拟文件系统，可以由不同的可插拔后端支持。后端支持以下文件系统操作：

| 工具 | 描述 |
| --- | --- |
| `ls` | 列出目录中的文件及其元数据（大小、修改时间） |
| `read_file` | 读取文件内容并附带行号，支持 offset/limit 用于大文件。还支持返回非文本文件（图像、视频、音频和文档）的多模态内容块。参见下面支持的扩展名。 |
| `write_file` | 创建新文件 |
| `edit_file` | 在文件中执行精确的字符串替换（具有全局替换模式） |
| `glob` | 查找匹配模式的文件（例如 `**/*.py`） |
| `grep` | 使用多种输出模式搜索文件内容（仅文件、带上下文的内容或计数） |
| `execute` | 在环境中运行 shell 命令（仅适用于 [sandbox 后端](https://docs.langchain.com/oss/python/deepagents/sandboxes)） |

支持的多模态文件扩展名

| 类型 | 扩展名 |
| --- | --- |
| [图像](https://docs.langchain.com/oss/python/langchain/messages#multimodal) | `.png`, `.jpg`, `.jpeg`, `.gif`, `.webp`, `.heic`, `.heif` |
| [视频](https://docs.langchain.com/oss/python/langchain/messages#multimodal) | `.mp4`, `.mpeg`, `.mov`, `.avi`, `.flv`, `.mpg`, `.webm`, `.wmv`, `.3gpp` |
| [音频](https://docs.langchain.com/oss/python/langchain/messages#multimodal) | `.wav`, `.mp3`, `.aiff`, `.aac`, `.ogg`, `.flac` |
| [文件](https://docs.langchain.com/oss/python/langchain/messages#multimodal) | `.pdf`, `.ppt`, `.pptx` |

虚拟文件系统被多个其他 harness 功能使用，例如 skills、memory、代码执行和上下文管理。您还可以在为 Deep Agents 构建自定义工具和中间件时使用文件系统。

更多信息，请参阅 [后端](https://docs.langchain.com/oss/python/deepagents/backends)。

## 文件系统权限

harness 支持声明式权限规则，用于控制 Agent 可以读取或写入哪些文件和目录。权限适用于上面列出的内置文件系统工具，并按声明顺序评估，采用首次匹配获胜的语义。

**工作原理：**

- 在创建 Agent 时将规则列表传递给 `permissions=`
- 每个规则指定 `operations`（`"read"`、`"write"`）、`paths`（glob 模式）和 `mode`（`"allow"` 或 `"deny"`）
- 第一个匹配的规则获胜。如果没有规则匹配，则允许该操作。

**为什么有用：**

- 将 Agent 限制在特定目录（例如 `/workspace/`）
- 保护敏感文件（例如 `.env`、凭据）
- 为子 Agent 提供比父 Agent 更窄的访问权限

权限不适用于 [sandbox 后端](https://docs.langchain.com/oss/python/deepagents/sandboxes)，后者支持通过 `execute` 工具执行任意命令。对于自定义验证逻辑，请使用 [后端策略挂钩](https://docs.langchain.com/oss/python/deepagents/backends#add-policy-hooks)。

有关完整的规则结构、示例和子 Agent 继承，请参阅 [权限](https://docs.langchain.com/oss/python/deepagents/permissions)。

## 任务委托（子 Agent）

harness 允许主 Agent 为隔离的多步骤任务创建临时的"子 Agent"。

**为什么有用：**

- **上下文隔离** - 子 Agent 的工作不会混乱主 Agent 的上下文
- **并行执行** - 多个子 Agent 可以并发运行
- **专业化** - 子 Agent 可以具有不同的工具/配置
- **Token 效率** - 大型子任务上下文被压缩为单个结果

**工作原理：**

- 主 Agent 有一个 `task` 工具
- 被调用时，它会创建一个具有自己上下文的新 Agent 实例
- 子 Agent 自主执行直到完成
- 向主 Agent 返回单个最终报告
- 子 Agent 是无状态的（不能发回多条消息）

**默认子 Agent：**

- "general-purpose" 子 Agent 自动可用
- 默认具有文件系统工具
- 可以使用额外的工具/中间件进行自定义

**自定义子 Agent：**

- 定义具有特定工具的专用子 Agent
- 示例：code-reviewer、web-researcher、test-runner
- 通过 `subagents` 参数配置

## 上下文管理

harness 管理上下文，以便深度 Agent 可以在 token 限制内处理长时间运行的任务，同时保留它们需要的信息。

**工作原理：**

- **输入上下文** — 系统提示、memory、skills 和工具提示塑造了 Agent 在启动时的知识
- **压缩** — 内置的卸载和总结在任务进展时保持上下文在窗口限制内
- **隔离** — 子 Agent 隔离繁重的工作并仅返回结果（参见 [任务委托](#task-delegation-subagents)）
- **长期内存** — 通过虚拟文件系统跨线程的持久存储

**为什么有用：**

- 启用超出单个上下文窗口的多步骤任务
- 保持最相关的信息在范围内，而无需手动修剪
- 通过自动总结和卸载减少 token 使用

有关配置详细信息，请参阅 [上下文工程](https://docs.langchain.com/oss/python/deepagents/context-engineering)。

## 代码执行

当您使用 [sandbox 后端](https://docs.langchain.com/oss/python/deepagents/sandboxes) 时，harness 会暴露一个 `execute` 工具，让 Agent 在隔离环境中运行 shell 命令。这使 Agent 能够安装依赖项、运行脚本并将代码执行作为其任务的一部分。

**工作原理：**

- Sandbox 后端实现 `SandboxBackendProtocolV2` — 检测到时，harness 会将 `execute` 工具添加到 Agent 的可用工具中
- 没有 sandbox 后端，Agent 只有文件系统工具（`read_file`、`write_file` 等）并且无法运行命令
- `execute` 工具返回合并的 stdout/stderr、退出代码，并截断大输出（保存到文件供 Agent 逐步读取）

**为什么有用：**

- **安全性** — 代码在隔离中运行，保护您的主系统免受 Agent 操作的影响
- **清洁环境** — 使用特定的依赖项或操作系统配置，无需本地设置
- **可重现性** — 跨团队一致的执行环境

有关设置、提供程序和文件传输 API，请参阅 [Sandbox](https://docs.langchain.com/oss/python/deepagents/sandboxes)。

## 人机协作

harness 可以在指定的工具调用处暂停 Agent 执行，以允许人工批准或修改。此功能通过 `interrupt_on` 参数可选启用。

**配置：**

- 将 `interrupt_on` 传递给 `create_deep_agent`，并带有工具名称到中断配置的映射
- 示例：`interrupt_on={"edit_file": True}` 在每次编辑前暂停
- 当提示时，您可以提供批准消息或修改工具输入

**为什么有用：**

- 破坏性操作的安全门
- 昂贵的 API 调用之前的用户验证
- 交互式调试和指导

## Skills

harness 支持为您的深度 Agent 提供专门工作流程和领域知识的 skills。

**工作原理：**

- Skills 遵循 [Agent Skills 标准](https://agentskills.io/)
- 每个 skill 是一个包含带有指令和元数据的 `SKILL.md` 文件的目录
- Skills 可以包括额外的脚本、参考文档、模板和其他资源
- Skills 使用渐进式披露——仅在 Agent 确定它们对当前任务有用时才加载
- Agent 在启动时读取每个 `SKILL.md` 文件的 frontmatter，然后在需要时查看完整的 skill 内容

**为什么有用：**

- 通过仅在需要时加载相关的 skills 来减少 token 使用
- 将功能捆绑到具有额外上下文的更大操作中
- 提供专业知识，而不会混乱系统提示
- 启用模块化、可重用的 Agent 功能

更多信息，请参阅 [Skills](https://docs.langchain.com/oss/python/deepagents/skills)。

## Memory

harness 支持持久化的内存文件，为您的深度 Agent 在对话之间提供额外的上下文。这些文件通常包含通用的编码风格、偏好、惯例和指导原则，帮助 Agent 理解如何与您的代码库协作并遵循您的偏好。

**工作原理：**

- 使用 [`AGENTS.md` 文件](https://agents.md/) 提供持久化上下文
- 内存文件总是被加载（不像使用渐进式披露的 skills）
- 在创建 Agent 时将一个或多个文件路径传递给 `memory` 参数
- 文件存储在 Agent 的后端（StateBackend、StoreBackend 或 FilesystemBackend）
- Agent 可以根据您的交互、反馈和识别的模式更新内存

**为什么有用：**

- 提供无需在每次对话中重新指定的持久化上下文
- 适用于存储用户偏好、项目指导原则或领域知识
- 始终可用于 Agent，确保一致的行为

有关配置详细信息和示例，请参阅 [Memory](https://docs.langchain.com/oss/python/deepagents/customization#memory)。
