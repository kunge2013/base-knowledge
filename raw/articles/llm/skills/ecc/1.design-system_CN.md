## 设计系统 — 生成与审查视觉系统

## 使用时机

- 启动需要设计系统的新项目
- 审查现有代码库的视觉一致性
- 在重新设计之前 — 了解现有内容
- 当 UI 看起来"不对劲"但找不出原因时
- 审查涉及样式的 PR

## 工作原理

### 模式 1：生成设计系统

分析代码库并生成一致的设计系统：

```
1. 扫描 CSS/Tailwind/styled-components 中的现有模式
2. 提取：颜色、排版、间距、圆角、阴影、断点
3. 研究 3 个竞品网站获取灵感（通过浏览器 MCP）
4. 提出设计令牌集（JSON + CSS 自定义属性）
5. 生成 DESIGN.md，包含每个决策的理由
6. 创建交互式 HTML 预览页面（自包含，无依赖）
```

输出：`DESIGN.md` + `design-tokens.json` + `design-preview.html`

### 模式 2：视觉审计

在 10 个维度上对 UI 进行评分（每个维度 0-10 分）：

```
1. 颜色一致性 — 是否在使用调色板，还是随机使用十六进制值？
2. 排版层次 — h1 > h2 > h3 > body > caption 是否清晰？
3. 间距节奏 — 是否使用一致的刻度（4px/8px/16px），还是随意设定？
4. 组件一致性 — 相似的元素是否看起来相似？
5. 响应式行为 — 在断点处是否流畅或出现破损？
6. 暗色模式 — 完整还是半成品？
7. 动画 — 有明确目的还是过度使用？
8. 可访问性 — 对比度、焦点状态、触摸目标
9. 信息密度 — 杂乱还是简洁？
10. 打磨程度 — 悬停状态、过渡效果、加载状态、空状态
```

每个维度都会给出评分、具体示例以及带有精确 file:line 的修复建议。

### 模式 3：AI 廉价设计检测

识别通用的 AI 生成设计模式：

```
- 到处滥用渐变
- 紫色到蓝色的默认渐变
- 无目的的"玻璃拟态"卡片
- 不该圆角的地方加了圆角
- 滚动时过度使用动画
- 通用型居中标题配渐变背景的 Hero 区域
- 毫无个性的无衬线字体栈
```

## 示例

**为 SaaS 应用生成设计系统：**

```
/design-system generate --style minimal --palette earth-tones
```

**审计现有 UI：**

```
/design-system audit --url http://localhost:3000 --pages / /pricing /docs
```

**检查 AI 廉价设计：**

```
/design-system slop-check
```
